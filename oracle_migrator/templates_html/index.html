Complexity Analysis

Upload Oracle Forms (.fmt, .xml) or Reports (.rdf, .xml) — individually, as a ZIP, or a mix — and get a complexity score from L1 (Trivial) to L5 (Very Complex) with per-trigger factor breakdowns and effort estimates.

Jakarta EE 10

Oracle Forms → complete Maven project: JPA entity, CDI service bean (@ApplicationScoped), EJB facade (@Stateless), JAX-RS REST endpoint, JSF backing bean + XHTML view. Deployable on WildFly / Payara.

JasperReports

Oracle Reports → full JRXML with all bands (Title, Header, Detail, Footer, Summary), migrated parameters, fields, and a Java runner class for PDF/XLSX export.

Supported Input
  • .fmt — Oracle Forms text export
  • .fmb / .xml — Oracle Forms XML export
  • .rdf / .xml — Oracle Reports XML export
  • .rpt — Oracle Reports text export
  • .zip — ZIP archive containing any of the above (folders included)
Complexity Scoring Factors
  • Trigger count & type weights (ON-INSERT/UPDATE/DELETE = weight 3)
  • DML inside triggers (INSERT / UPDATE / DELETE / MERGE)
  • Dynamic SQL (EXECUTE IMMEDIATE / EXEC_SQL)
  • Explicit cursors, loops, CASE/ELSIF expressions
  • Exception handling blocks
  • Navigation calls (GO_BLOCK / GO_ITEM)
  • Program units & SQL query count

Complexity Levels

Every file is assigned one of five levels. Oracle Forms are scored on trigger complexity; Oracle Reports use the Re_Forms 21 metric model (validated against 87 real reports).

L1 — Trivial

Very few triggers · No DML · No cursors · Minimal PL/SQL · Reports score < 25

Effort: < 1 day

L2 — Simple

A few triggers · Basic validation · Light exception handling · Reports score 25–49

Effort: 1–3 days

L3 — Moderate

DML in triggers · Explicit cursors or loops · Navigation calls · Reports score 50–74

Effort: 5–10 days

L4 — Complex

Dynamic SQL · ON-INSERT / UPDATE / DELETE · Many program units · Reports score 75–99

Effort: 10–20 days

L5 — Very Complex

High trigger density · EXECUTE IMMEDIATE · Extensive PL/SQL · Reports score ≥ 100

Effort: 20+ days
